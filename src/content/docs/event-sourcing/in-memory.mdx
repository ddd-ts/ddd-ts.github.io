title: "@ddd-ts/event-sourcing-inmemory"
description: In-memory event streams, lakes and projected stream readers, for tests and local runs.
---

The in-memory event sourcing adapter is the reference implementation of every
contract defined in `@ddd-ts/core`. It uses plain JavaScript arrays as the
backing storage, runs synchronously, and is the fastest way to exercise an
event-sourced domain in a test suite.

## Installation

```sh
pnpm add -D @ddd-ts/event-sourcing-inmemory
```

## Event streams

```ts
import {
  InMemoryEventStreamStore,
  InMemoryEventStreamStorageLayer,
  MakeInMemoryEventStreamAggregateStore,
} from "@ddd-ts/event-sourcing-inmemory";

const storage = new InMemoryEventStreamStorageLayer();
const eventStore = new InMemoryEventStreamStore(storage, serializers);

const AccountStore = MakeInMemoryEventStreamAggregateStore(Account);
const accounts = new AccountStore(eventStore);
```

`InMemoryEventStreamStorageLayer` is a plain in-memory map of streams. You
can construct multiple stores on top of the same layer to test cross-stream
behaviour.

## Event lakes

```ts
import {
  InMemoryEventLakeStorageLayer,
  InMemoryEventLakeStore,
  MakeInMemoryEventLakeAggregateStore,
} from "@ddd-ts/event-sourcing-inmemory";

const storage = new InMemoryEventLakeStorageLayer();
const lake = new InMemoryEventLakeStore(storage, serializers);
```

`InMemoryEventLakeStore` appends events to a single ordered log, useful for
projections that need cross-aggregate ordering.

## Projected stream reader

```ts
import {
  InMemoryProjectedStreamStorageLayer,
  InMemoryProjectedStreamReader,
  InMemoryStreamSourceFilter,
  InMemoryLakeSourceFilter,
} from "@ddd-ts/event-sourcing-inmemory";

const reader = new InMemoryProjectedStreamReader(
  new InMemoryProjectedStreamStorageLayer(streamStorage, lakeStorage),
);

for await (const event of reader.read(projectedStream)) {
  await projection.handle(event);
}
```

## Snapshots

```ts
import { InMemorySnapshotter } from "@ddd-ts/event-sourcing-inmemory";

const snapshots = new InMemorySnapshotter();
const AccountStoreWithSnapshots = MakeInMemoryEventStreamAggregateStore(Account, {
  snapshotter: snapshots,
  every: 50, // snapshot every 50 events
});
```

The snapshotter is plugged into the aggregate store. It persists serialized
aggregate state every *N* events and reads from the latest snapshot first on
load, then tails the remaining events.

## Use it as a faithful test double

The in-memory adapter is what the shared `EventStreamStoreSuite`,
`EventLakeStoreSuite` and `ProjectedStreamReaderSuite` are run against. The
Firestore adapter is run against the *same* suite. So a passing test against
the in-memory adapter is a strong (though not absolute) signal that the
production adapter will behave the same way.
